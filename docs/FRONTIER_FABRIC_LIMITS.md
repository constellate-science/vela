# Universality and limits

## 1. What is universal

There are two distinct universality claims.

### Representation universality

The fabric applies to finite positive ranked dependency systems whose domain
artifacts can be compiled into:

```text
claim-context-polarity cells
primitive assumption atoms
alternative and joint support routes
active-view substitutions
root-bound observations
```

This covers a broad class of exact, computational, experimental, statistical,
and reviewed scientific records without claiming that their evidence is
interchangeable.

### Extension universality

Every domain may expose candidates, obligations, receipts, and transfers through
the same adapter boundary. The degree of automation is domain-dependent.

```text
exact / replay domains     verifier-gated search can close the loop in software
trace / estimate domains   software validates receipts and analysis, not nature itself
attestation domains        eligible humans remain the evidence source
```

The autonomous discovery loop retains the cheap-verifier boundary. Universal
representation is not universal permissionless verification.

## 2. What remains domain-specific

The following remain outside the universal core:

```text
what counts as a claim
which contexts matter
which artifacts are admissible
what a verifier or trace establishes
which obligations define a useful frontier
which transfers are scientifically licensed
which utilities rank next actions
which safety and access policies apply
```

Adapters carry these differences.

Independent adapters are conservative by construction: adding a fresh domain
profile changes no existing lineage. Cross-domain consequences require explicit
accepted bridges. This modularity, rather than a shared global ontology, is what
makes extension across domains tractable.

## 3. No universal truth scalar

The fabric does not collapse proof, replay, observation, estimate, and attestation into one score. It can project each into product views while retaining the underlying class and lineage.

## 4. No universal ontology requirement

Global ontology agreement is not required before use. Cells use domain-local schemas and explicit bridge objects. Common identifiers and ontologies may improve discovery but do not define state identity by themselves.

## 5. No claim to complete science

The system can map all frontiers that participating adapters and institutions declare. It cannot certify that every scientifically meaningful question has been declared.

## 6. Recursion, negation, aggregation, and causal transport

The kernel remains finite, positive, and ranked.

```text
negation          represented by positive evidence for a negative-polarity claim
aggregation       performed by proof-carrying evaluators over accepted inputs
recursion         compiled or handled outside the core until a real cyclic producer requires it
causal transport  remains a typed target-checked bridge with explicit causal assumptions
```

These boundaries preserve replay and prevent the universal core from becoming an uncheckable scientific logic.

## 7. The strongest honest claim

Vela and Constellate can become a common operating layer for scientific frontier state if independent domain tools compile their accepted results, gaps, candidates, corrections, and transfers through this kernel.

The architecture makes that possible. Adoption determines whether it becomes common.
